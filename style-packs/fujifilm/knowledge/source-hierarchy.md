# Source Hierarchy

FujiDay uses a strict two-layer source model.

## Tier 1: Official Fujifilm definition layer

Use official Fujifilm materials to define:

- what each Film Simulation is
- what recipe settings do
- what workflows Fujifilm supports
- where compatibility is conditional

Official semantics win whenever community lore differs.

## Tier 2: Community case-study layer

Use community recipe libraries such as Fuji X Weekly to study:

- repeated parameter patterns
- naming conventions
- recipe families
- body-generation compatibility habits

Community recipes are examples, not canonical definitions.
